La value function definisce la bontà (i.e., l'[expected return](reward.md#Expected%20return)) di uno stato $x$ data una data [policy](reinforcement%20learning/policy.md) $\pi$ come $V_\pi(x)=\mathbb{E}\{\displaystyle\sum_{\tau=t}^\infty\gamma^{\tau-t}r(x(\tau),\pi(x(\tau)))\}$. È quindi un vettore della dimensionalità degli stati, dove nella posizione $i$-esima c'è il valore dell'$i$-esimo stato se seguiamo la policy $\pi$.
Si può calcolare usando l'[equazione di Bellman](equazione%20di%20Bellman.md).
# Relazione con expected return
Se ci mettiamo nel caso $t=0$, stiamo partendo dallo stato iniziale $x(t)=x(0)$ e allora la value function coincide con l'[expected return](reward.md#Expected%20return): $R_\pi=\mathbb{E}\left\{\displaystyle\sum_{\tau=0}^\infty\gamma^{\tau}r(x(t),\pi(x(t))\right\}$, cioè $V_\pi(x(0))=R_\pi$.
Vogliamo valutare la policy in modo robusto, cioè in modo che il valore legato a uno stato sia lo stesso a prescindere dallo stato iniziale di partenza. Questo è possibile per [condizione di markov](condizione%20di%20markov.md), per cui il futuro dipende solo dal presente e non dal passato; questo vuol dire che il solo stato attuale $x(t)$ raccoglie le informazioni del passato.
Possiamo a questo punto vedere la value function di uno stato come il la media dell'expected return che otteniamo eseguendo l'esperimento molte volte a partire da quello stato e a partire da tutti i possibili istanti di tempo in cui quello stato viene visitato. Questo spiega anche come la value function sia time indipendent, cioè come il valore legato a uno stato non pendenda da quando entriamo in quello stato.
